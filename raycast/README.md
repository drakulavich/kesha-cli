# Kesha Voice Kit — Raycast extension

Offline speech-to-text and text-to-speech for Raycast, powered by the [kesha-voice-kit](https://github.com/drakulavich/kesha-voice-kit) CLI. No API keys, no cloud — runs locally on Apple Silicon.

## Commands

### Transcribe Selected Audio

Select an audio file in Finder (`.ogg`, `.opus`, `.mp3`, `.m4a`, `.wav`, `.flac`, `.webm`), then run this command. Opens a detail view with the transcript and detected language; the transcript is already on your clipboard.

### Speak Clipboard

Synthesize speech from the current clipboard text and play it through the default audio output. Auto-routes English → Kokoro and Russian → Piper; override with the _Default voice_ preference (any valid `kesha say --voice` value including `macos-*` system voices).

## Prerequisites

Install the `kesha` CLI globally and fetch the engine + models:

```bash
bun add --global @drakulavich/kesha-voice-kit
kesha install          # ASR + lang-id (~2.6 GB)
kesha install --tts    # Kokoro + Piper TTS models (~390 MB, required by Speak Clipboard)
```

`kesha install --tts` also needs `espeak-ng` on your PATH:

```bash
brew install espeak-ng
```

macOS `macos-*` system voices need no install — they use voices already on your Mac.

## Preferences

| Preference | Default | When to set |
|---|---|---|
| `kesha` binary path | empty (uses PATH) | If `kesha` is installed somewhere PATH doesn't cover in Raycast's subprocess env |
| Default voice | empty (auto-route) | If you want a specific voice for _Speak Clipboard_, e.g. `macos-com.apple.voice.compact.en-US.Samantha` |

## Development

```bash
cd raycast
bun install        # or npm install
bun run dev        # starts `ray develop` against your local Raycast
```

`ray develop` registers the extension in Raycast; editing files hot-reloads. Commit the built output by NOT checking in `dist/` — it's generated by `ray build` at publish time.

## Publishing (maintainer-only)

```bash
cd raycast
bun run build
bun run publish    # opens a PR against raycast/extensions
```
